1. THE PAYMENT & WALLET ECOSYSTEM
-------------------------------------------------------------------------------

The Murranno payment system is a professional-grade fintech implementation that 
bridges the gap between global music streaming royalties and localized Nigerian 
payouts. It is built on a "Zero-Trust" architecture.

A. LEDGER VS. VAULT (The Separation of Concerns)
   - THE LEDGER (Supabase/Database): This is the system of record. It tracks 
     "Digital Debt." Every ₦1 shown in an artist's wallet is a record that 
     Murranno owes them that amount.
   - THE VAULT (Paystack/Corporate Accounts): This is where the physical cash 
     resides. Incoming royalties are settled into Murranno's corporate accounts, 
     while outgoing payments are moved through the Paystack Merchant Balance.

B. WALLET BALANCES
   - Available Balance: Funds that have cleared all holding periods and are 
     ready for instant or reviewed withdrawal.
   - Pending Balance: Funds that are currently in a "clearing" or "in-transit" 
     state (e.g., royalties reported but not yet settled, or withdrawals 
     awaiting approval).
   - Total Earnings: A lifetime cumulative tally of every kobo generated by 
     the artist's catalog.

C. THE HARDENED LEDGER (Ledger Integrity)
   - Chained Hashing: Every transaction in the ledger contains a 'hash' and a 
     'previous_hash'. Like a blockchain, each row is cryptographically linked 
     to the one before it. If a database administrator attempts to manually 
     change an amount, the chain breaks instantly.
   - Append-Only Rule: Database triggers prevent any 'UPDATE' or 'DELETE' on 
     transaction rows. The only way to correct a mistake is to issue a new 
     'Adjustment' transaction.
   - Reconciliation RPC: An automated server-side process that periodically 
     sums all ledger entries and verifies that the total matches the 
     'wallet_balance' table exactly.

D. THE WITHDRAWAL PIPELINE
   1. Initiation: Artist requests withdrawal via the mobile app.
   2. Idempotency Check: A unique key prevents the system from processing the 
      same withdrawal twice if the user taps the button multiple times.
   3. Security Gating:
      - Transaction PIN: Verified via bcrypt hashing.
      - Payout Lock: Prevents withdrawals for 24-48 hours after a password 
        reset or bank account change.
      - Velocity Limits: Capped at ₦500,000 per 24 hours to prevent massive 
        drainage in case of account compromise.
      - OTP Verification: High-value withdrawals (>₦100,000) require a 6-digit 
        code sent to the artist's email.
   4. Anomaly Detection: The system tracks the IP address and Device ID. If the 
      withdrawal attempt looks significantly different from the artist's 
      normal pattern, it is auto-flagged for review.
   5. Tiered Settlement:
      - Tier 1 (<₦5,000): Instant transfer via Paystack.
      - Tier 2 (₦5,000 - ₦50,000): 12-hour automated cooling-off period.
      - Tier 3 (>₦50,000): Mandatory manual approval by a Murranno Admin.

2. THE MUSIC DISTRIBUTION SYSTEM
-------------------------------------------------------------------------------

The distribution system is the core engine that moves music from the artist's 
phone to over 150 global streaming platforms (DSPs) like Spotify, Apple Music, 
and Audiomack.

A. RELEASE CREATION & METADATA
   - Metadata Management: Every release (Single/Album) captures critical 
     information: Title, Genre, Label Name, Release Date, and Language.
   - UPC/EAN Generation: The system automatically assigns a Universal Product 
     Code to every album/single for global tracking.
   - ISRC Generation: Every individual track is assigned an International 
     Standard Recording Code, which is the "Digital Fingerprint" used to track 
     streams and royalties.

B. ASSET HANDLING (High-Fidelity Pipeline)
   - Audio Processing: Artists upload high-quality audio files. The Edge 
     Functions use chunked processing to handle large files (WAV/FLAC) without 
     crashing.
   - Cloudinary Integration: Cover art and audio are stored on Cloudinary with 
     signed URLs. This ensures that assets are secure and delivered via 
     High-Speed CDNs (Content Delivery Networks) globally.
   - Image Optimization: Cover art is automatically resized and optimized for 
     DSPs while maintaining professional quality.

C. THE DISTRIBUTION WORKFLOW
   1. Submission: Artist submits the release.
   2. Moderation (Admin Review): Murranno admins review the release for 
      copyright compliance and metadata accuracy.
   3. Packaging: The system packages the audio and metadata into the XML 
      formats required by the DDEX (Digital Data Exchange) standard.
   4. Delivery: The assets are pushed to the DSPs.
   5. Go-Live: The music appears on Spotify, Apple Music, etc., on the 
      specified release date.

D. ROYALTY INGESTION
   1. Reporting: DSPs send "Sales Reports" containing millions of rows of 
      streaming data.
   2. Parsing: Murranno's backend parses these reports, identifying which 
      streams belong to which artist based on the ISRC.
   3. Allocation: The net earnings (after store and platform commissions) are 
      calculated and automatically credited to the artist's wallet.
   4. Analytics: This data is then visualized in the artist's dashboard, 
      showing streams by country, platform, and date.

3. SUBSCRIPTION & MONETIZATION SYSTEM
-------------------------------------------------------------------------------

Murranno uses a hybrid monetization model combining Software-as-a-Service (SaaS) 
subscriptions and distribution commissions.

A. SUBSCRIPTION TIERS
   - Artist (Free/Basic): Standard distribution with a higher platform commission.
   - Pro (Monthly/Yearly): Lower commission, faster distribution, and basic 
     analytics.
   - Label: Advanced tools for managing multiple artists under one account, 
     custom label name on stores, and priority support.
   - Agency: The highest tier, offering full catalog management, bulk uploads, 
     and dedicated account management.

B. PAYMENT LIFECYCLE
   1. Initialization: The app calls `paystack-initialize-subscription`. This 
      sets up a transaction on Paystack and returns a payment URL.
   2. Payment: The artist pays via Card, Bank Transfer, or USSD.
   3. Webhook Fulfillment: Paystack sends a secure POST request to 
      `paystack-subscription-webhook`. The function:
      - Validates the Paystack signature to prevent spoofing.
      - Identifies the user via metadata.
      - Updates the `subscriptions` table with the new tier and expiry date.
      - Sends a success notification to the artist.

C. REVENUE MODEL
   - Distribution Commission: Murranno retains a small percentage (e.g., 10-30% 
     depending on tier) of all royalties collected from DSPs.
   - Subscription Fees: Recurring revenue from Pro, Label, and Agency users.
   - Promotion Fees: One-time payments for marketing campaigns.

4. CAMPAIGN & PROMOTION ENGINE
-------------------------------------------------------------------------------

Artists can use their earnings or external payments to boost their music 
visibility through the integrated campaign manager.

A. PROMOTION TYPES
   - TikTok Viral Push: Coordination with influencers to create sound-based 
     challenges.
   - Spotify Playlist Pitching: Direct pitching to editorial and independent 
     curators.
   - Instagram Reels Promo: Targeted visual content promotion.

B. BUDGET & METRICS TRACKING
   - Real-time Spend: Artists can see exactly how much of their budget has been 
     spent in the `campaign_metrics` table.
   - Performance Data: The system tracks Impressions, Clicks, Conversions, and 
     Engagement rates, allowing artists to see the ROI of their marketing 
     spend.

5. AUTHENTICATION & SECURITY HARDENING
-------------------------------------------------------------------------------

A. IDENTITY MANAGEMENT (Supabase Auth)
   - JWT (JSON Web Tokens): Every request from the mobile app is signed with a 
     unique token that expires frequently.
   - Row Level Security (RLS): The database itself enforces security. Even if 
     a user manages to access the DB, they can only "SEE" and "EDIT" rows where 
     `user_id = auth.uid()`.

B. TRANSACTION PIN ARCHITECTURE
   - Bcrypt Hashing: Transaction PINs are never stored in plain text. They are 
     hashed using bcrypt with a high "Work Factor," making it mathematically 
     impossible for even Murranno staff to know an artist's PIN.
   - Cold Storage: Sensitive ID documents (NIN slips, Passports) are stored in 
     Private Cloudinary buckets with expiring access tokens.

C. MAINTENANCE & SERVICE CONTROL
   - Maintenance Mode: Admins can toggle a global flag to pause payouts or 
     uploads during system upgrades, preventing data corruption.
   - Audit Logs: Every administrative action (Approving a release, Rejecting a 
     payout, Changing a user role) is logged with the Admin ID, Timestamp, 
     Action Type, and IP Address.

-------------------------------------------------------------------------------
6. ANALYTICS & DATA ARCHITECTURE
-------------------------------------------------------------------------------

The Analytics system turns millions of raw streaming data points into 
actionable insights for the artist.

A. DATA INGESTION
   - The `streaming_data` table stores daily stream counts broken down by 
     Track ID, Platform (Spotify, Apple, etc.), and Country.
   - This data is updated as soon as DSP reports are ingested (usually daily 
     or weekly).

B. BREAKDOWN CAPABILITIES
   - Platform Insights: See which store is generating the most revenue.
   - Geographic Insights: Identify which countries (e.g., Nigeria, USA, UK) 
     are the top listeners.
   - Growth Metrics: Automated calculation of Week-over-Week (WoW) or 
     Month-over-Month (MoM) growth.

-------------------------------------------------------------------------------
7. EDGE FUNCTIONS: THE PLATFORM "BRAIN"
-------------------------------------------------------------------------------

All logic is executed via Supabase Edge Functions (Deno/TypeScript), which 
provides the following benefits:
- Security: Secrets (like Paystack/Cloudinary keys) never touch the mobile app.
- Performance: Functions run globally in data centers close to the user.
- Integration: Direct, secure communication with Paystack (Payments), 
  Cloudinary (Media), and Resend (Emails).

CRITICAL FUNCTIONS:
- paystack-initiate-withdrawal: The security gate for all payouts.
- admin-review-withdrawal: The tool for Murranno staff to approve large sums.
- upload-audio-cloudinary: Handles high-bandwidth music uploads.
- get-earnings-breakdown: Aggregates millions of stream rows into simple data.
- paystack-subscription-webhook: Automatically upgrades accounts upon payment.
